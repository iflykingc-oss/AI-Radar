# AI Radar — 产品发现平台深度调研报告

> 调研目标：分析 Product Hunt、BetaList、Futurepedia、There's An AI For That、G2、Capterra 等主流产品发现平台的产品详情页、发现过滤、对比功能、社交信任信号、数据可视化等设计模式
> 日期：2026-05-29
> 方法：Web 搜索 + Web Fetch + GitHub 开源仓库分析

---

## 一、产品详情页（Product Detail Pages）

### 1.1 Product Hunt

**信息密度：极高，但聚焦社区互动**

| 模块 | 内容 |
|------|------|
| **头部区** | 产品 Logo、名称、一句话简介（tagline）、官网链接 |
| **互动信号** | 投票数（Upvotes）带计数徽章、评论数、收藏（Bookmark） |
| **媒体画廊** | 多张截图轮播（1270x760 标准尺寸）、支持视频嵌入 |
| **Maker 信息** | 创始人/团队成员头像 + 链接，支持 Maker 自我介绍 |
| **话题/分类** | Topics 标签（如 #AI #Developer Tools #Productivity）可点击跳转 |
| **讨论区** | 嵌套评论系统，支持回复、点赞，Maker 有专属标识 |
| **相关产品** | 下方推荐 "More from this maker" 和 "Similar products" |
| **社交证明** | 投票数排名（#1 Product of the Day）、评论者身份认证 |

**关键设计模式：**
- 投票计数是核心社交信号，页面顶部大数字展示制造紧迫感
- 截图画廊遵循 "5 帧故事框架"：问题 -> 解决方案 -> 核心功能 -> 使用场景 -> CTA
- Maker 亲自参与评论区回答，建立产品信任

### 1.2 G2

**信息密度：极高，B2B 专业级**

| 模块 | 内容 |
|------|------|
| **评分总览** | 综合评分（星级）、评价数量、G2 Score（算法评分）、满足度 |
| **雷达图** | 多维度雷达图/蜘蛛图：易用性、功能、支持、设置、质量/价值 |
| **用户评论** | 结构化评论列表：标题、正文、优点、缺点、评分、用户身份验证标记 |
| **功能矩阵** | 详细功能清单，分类展示（核心功能、高级功能、集成能力） |
| **定价信息** | 定价模式（免费试用/免费增值/按量付费/企业定制）、联系销售按钮 |
| **替代品** | "Alternatives" 模块列出竞品，可一键跳转到竞品详情页 |
| **市场分类** | 产品在多个 G2 Grid（象限图）中的位置展示 |
| **集成列表** | 支持与哪些其他产品的集成，按类别分组 |

**关键设计模式：**
- **G2 Grid 象限图**：将产品放在 Leader/High Performer/Contender/Niche 四个象限，视觉化竞争格局
- **结构化评论**：强制要求用户填写 "优点"、"缺点"、"使用时长"、"角色/行业"，评论质量高
- **评分维度细分**：不只一个总分，而是 5-6 个维度的独立评分

### 1.3 Capterra

**信息密度：高，侧重比较**

| 模块 | 内容 |
|------|------|
| **评分卡片** | 星级评分、评价数、推荐百分比（% 会推荐给朋友） |
| **功能标签** | Top Features 列表，3-5 条核心功能高亮 |
| **定价** | Starting Price（起步价）、定价模型、免费版可用性 |
| **评论精选** | Top Review 摘录，带 "Read All Reviews" 跳转 |
| **对比入口** | "Compare with Similar Products" 按钮，跳转到对比页 |
| **供应商信息** | 公司名称、成立年份、总部位置、员工规模 |

**关键设计模式：**
- **推荐百分比**是独特的信任信号（"95% would recommend"），比纯星级更有说服力
- **起步价格醒目展示**在头部，降低决策摩擦
- 评论支持按 "角色"、"公司规模"、"使用时长" 筛选

### 1.4 BetaList

**信息密度：中等，聚焦早期项目**

| 模块 | 内容 |
|------|------|
| **基本信息** | Logo、名称、Tagline（一句话描述）、官网链接 |
| **状态标签** | Pre-launch / Beta / Live |
| **分类** | 垂直领域标签（SaaS, Analytics, AI Tools, Developer Tools 等） |
| **时间信息** | 发布日期（按天分组展示） |
| **热度标识** | Trending Startups 专区、BOOSTED 标记（付费推广） |
| **邮件订阅** | 每封邮件推送当日新项目的入口 |

**关键设计模式：**
- **时间轴分组**：项目按日期分组（Today, Yesterday, May 27th），制造持续更新感
- 极简卡片设计，信息密度低但扫描效率高
- 无评论/评分系统，定位是 "发现早期项目" 而非 "评估成熟产品"

### 1.5 Futurepedia

**信息密度：高，AI 工具专精**

| 模块 | 内容 |
|------|------|
| **分类体系** | 多级分类（Writing, Image & Video, Coding, Marketing, Audio, Productivity 等 50+ 类别） |
| **定价标签** | Free / Freemium / Paid / Contact for Pricing 四色 Badge |
| **平台标签** | Web, iOS, Android, Chrome Extension, Slack, API 等平台兼容性 |
| **验证状态** | Verified 徽章、Open Source 标记 |
| **功能标签** | 2-4 个核心场景 Tag，可点击跳转筛选 |
| **评分/收藏** | 星级评分、收藏/保存数（saves） |
| **元数据** | Last Updated 时间、Added Date、开发者/公司信息 |
| **社交媒体** | X (Twitter)、LinkedIn、GitHub、Discord 链接 |
| **替代品/推荐** | Alternatives 模块、More in [Category] 推荐 |

**关键设计模式：**
- **漏斗式信息布局**：身份识别 -> 功能说明 -> 视觉验证 -> 定价透明 -> 社交背书 -> 关联发现
- 定价模式用色块 Badge 区分，一眼可辨
- 平台兼容性标签让用户快速判断是否匹配自己的使用环境

### 1.6 There's An AI For That (TAAFT)

**信息密度：高，AI 任务导向**

| 模块 | 内容 |
|------|------|
| **任务导航** | 按使用场景/任务分类（"What do you need help with?"） |
| **搜索** | 语义搜索 + 自动补全 |
| **Leaderboard** | AI 工具排行榜（按热度/评分排序） |
| **产品卡片** | Logo、名称、一句话描述、定价模式、发布时间 |
| **时间线** | 按 AI 模型/技术发布的时间线展示 |
| **Deals** | 专属优惠/折扣入口 |

**关键设计模式：**
- **任务导向导航**：不是按工具分类，而是按用户"需要什么帮助"分类，降低认知负荷
- **Leaderboard 排名系统**：制造竞争感和发现动力
- **时间线视图**：展示 AI 工具随技术演进的发布节奏

## 二、发现/过滤 UX（Discovery & Filtering）

### 2.1 高级筛选模式对比

| 平台 | 筛选维度 | 特色能力 |
|------|----------|----------|
| **Product Hunt** | 话题（Topics）、分类（Categories）、时间范围、排序 | 每日/每周/每月榜单，话题云 |
| **G2** | 行业、公司规模、部署类型、定价、功能、评分、地区 | 智能推荐引擎（基于角色和需求问卷） |
| **Capterra** | 行业、部署类型、定价模型、功能需求 | 向导式筛选（回答 3-5 个问题后推荐） |
| **Futurepedia** | 定价（Free/Freemium/Paid）、分类（50+）、平台（Web/iOS/API）、开源状态 | 多维度交叉筛选，价格滑块 |
| **BetaList** | 分类标签栏（12 个垂直领域） | 极简，主要靠时间轴和分类发现 |
| **TAAFT** | 任务/场景搜索、定价、分类 | 语义搜索 + 任务导航 |

### 2.2 可复用模式

1. **分类 + 定价交叉筛选**（Futurepedia/TAAFT）：用户先选"我要做什么"，再选"我愿意付多少钱"
2. **向导式推荐**（G2/Capterra）：通过 3-5 个问题引导用户（"你的团队规模？" "需要什么功能？"），返回个性化推荐
3. **话题云/标签系统**（Product Hunt）：高频话题以大字号显示，制造趋势感
4. **平台兼容性过滤**（Futurepedia）：按 Web/iOS/Android/Chrome Extension/API 过滤，AI 场景非常实用
5. **语义搜索**（TAAFT）：支持自然语言查询（"help me write blog posts"），返回匹配工具

## 三、对比功能（Comparison Features）

### 3.1 对比模式分析

| 平台 | 对比方式 | 特色 |
|------|----------|------|
| **G2** | 并排对比（Side-by-Side） | 最多 4 产品并排，每个维度独立评分 + 雷达图叠加 |
| **Capterra** | 对比表（Comparison Table） | 功能矩阵对比（行=功能，列=产品），有/无/部分支持三色标记 |
| **Product Hunt** | 无原生对比 | 评论区用户自发对比 |
| **Futurepedia** | 无原生对比 | 替代品列表 + 分类内推荐 |
| **BetaList** | 无 | 无对比需求（早期项目发现） |

### 3.2 G2 对比页详解

- **表头固定**：对比产品卡片固定在顶部，滚动时保持可见
- **评分维度网格**：每个产品在 5-6 个维度上独立评分，支持柱状图/雷达图可视化
- **功能矩阵**：逐行对比每个功能点，使用 ?/??/? 三态标记
- **用户评论对比**：每个产品的 Top 评论并列展示
- **价格对比**：起步价、定价模型、免费版功能差异一目了然
- **用户画像对比**：各产品的用户公司分布、角色分布

### 3.3 Capterra 对比页详解

- **功能矩阵表格**：50+ 功能行 × N 产品列，支持过滤"只看我有需求的功能"
- **评分对比**：星级 + 推荐百分比 + 评价数并列
- **优缺点并列**：每个产品的 Top Pros/Cons 并排展示
- **用户反馈摘要**：自动生成 "Users say..." 段落（从评论中提取共性观点）

## 五、数据可视化（Data Visualization）

### 5.1 可视化模式清单

| 可视化类型 | 平台 | 用途 |
|------------|------|------|
| **雷达图/蜘蛛图** | G2 | 多维度评分叠加对比 |
| **象限图（G2 Grid）** | G2 | 产品市场位置（满意度 vs 市场份额） |
| **柱状图** | G2/Capterra | 各维度评分对比 |
| **星级评分条** | 所有平台 | 快速评分展示 |
| **趋势/增长率指示器** | TAAFT（Leaderboard） | 热度排名变化 |
| **时间轴** | BetaList | 项目发布日期分组 |
| **话题云** | Product Hunt | 热门话题词频可视化 |
| **词云** | — | AI Radar 已实现（/trends 页） |
| **嵌入 SVG 图表** | star-history | README 可嵌入的 star 历史曲线 |

### 5.2 值得借鉴的可视化设计

1. **G2 Grid 象限图**：横轴 = 市场存在感（评论数/引用数），纵轴 = 用户满意度。四个象限：Leader / High Performer / Contender / Niche。**AI Radar 可以按"置信度 vs 活跃度"做类似象限**
2. **雷达图多维评分**：AI Radar 的四维验证（Freshness / Multi-source / Engagement / Technical）天然适合雷达图展示
3. **Star 历史曲线**（star-history）：GitHub star 随时间变化曲线，展示增长趋势
4. **话题云**（Product Hunt）：高频话题词放大显示，低频缩小，制造趋势感
5. **Leaderboard 排名**（TAAFT）：Top N 产品排行榜 + 排名变化箭头（上升/下降）

## 六、开源实现参考

### 6.1 OpenAlternative（GitHub: kymokleo/OpenAlternative）

**技术栈**：Next.js App Router + TypeScript + Prisma + Turborepo Monorepo
**架构特点**：
- Monorepo 分层：`/apps/app`（主业务）、`/apps/analyzer`（数据分析）、`/packages/db`（数据库 Schema）、`/packages/github`（GitHub API 集成）
- Server Actions 处理表单提交和数据变更
- Inngest 后台任务（截图生成、数据同步）
- Stripe 支付 + Beehiiv 邮件订阅
- AWS S3 文件存储 + ScreenshotOne 产品截图

**数据结构（推断）**：
- `Product/Alternative`：名称、描述、官网、GitHub 链接、许可证、分类/标签、活跃度指标
- `Category/Tag`：分类体系，多对多关联
- 定期同步 GitHub Star/Fork/Commit 数据

### 6.2 star-history（GitHub: star-history/star-history）

**技术栈**：Next.js + 独立后端 API
**核心能力**：
- GitHub Star 时间序列追踪
- 生成可嵌入的 SVG 图表（`/chart?repos=...`）
- Chrome 扩展
- 21k+ stars

### 6.3 GitHub-Trending-API（GitHub: NiklasTiede）

**技术栈**：FastAPI + Pydantic + Docker
**数据 Schema**：
```json
{
  "rank": 1,
  "username": "sherlock-project",
  "repositoryName": "sherlock",
  "description": "...",
  "language": "Python",
  "totalStars": 21977,
  "forks": 2214,
  "starsSince": 462,
  "since": "daily",
  "builtBy": [{"username": "...", "url": "...", "avatar": "..."}]
}
```

## 九、关键结论

1. **信息漏斗式设计**是所有成功平台的共性：快速识别 -> 深入了解 -> 建立信任 -> 行动引导
2. **多维评分 > 单一评分**：G2 的雷达图、Capterra 的推荐百分比都证明多维度评估更有说服力
3. **验证标记是 AI 工具目录的核心信任要素**：AI 领域信息更新极快，Last Updated + Verified 是建立信任的关键
4. **替代品推荐形成站内流量闭环**：用户看完一个产品后，80%+ 会继续浏览替代品或相关产品
5. **任务导向导航 > 工具分类导航**：TAAFT 证明按"用户需要什么帮助"分类比按"工具类型"分类更符合用户心智模型
6. **可视化竞争格局制造稀缺感**：G2 Grid 象限图、Leaderboard 排名都能驱动用户参与
